// OVERVIEW

Two systems,
one defender's edge.

HoneyPot Wars is built as two cooperating systems. A platform handles client accounts, consented scan requests, admin approval, and the secure delivery of reports. A separate autonomous AI agent performs the assessment itself — reasoning over a target like an attacker, then writing up what it found.

The whole thing is secure by construction: row-level security, server-only admin actions, and reports that are never publicly reachable.

// SECURE BY CONSTRUCTION

The boring parts done right.

🔐
Row-level security

Every client is locked to their own profile, requests and reports — enforced in the database.

🛡️
Server-only admin

Admin actions run behind an email allowlist in middleware and API routes, using a service-role key.

🚫
No self-activation

A DB trigger blocks any non-admin from changing their own account status.

📂
Private report bucket

Reports live in a private bucket, readable only by the owning client or an admin.
